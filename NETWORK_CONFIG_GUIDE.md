# 公司WiFi IP地址配置指南

## 如何找到公司WiFi的IP地址段

### 方法1：在已连接公司WiFi的设备上查看（最简单）

#### Windows系统：
1. 按 `Win + R`，输入 `cmd`，按回车
2. 在命令提示符中输入：`ipconfig`
3. 找到"无线局域网适配器 WLAN"部分
4. 查看 **IPv4 地址**，例如：`192.168.1.105`
5. 查看 **默认网关**，例如：`192.168.1.1`

**判断IP段：**
- 如果IP是 `192.168.1.105`，网关是 `192.168.1.1`
- 那么IP段通常是 `192.168.1.0/24`（覆盖 192.168.1.1 到 192.168.1.254）

#### Mac系统：
1. 打开"系统设置" → "网络"
2. 选择WiFi连接
3. 点击"详细信息"
4. 查看 **IP地址** 和 **路由器** 地址

#### 手机（Android/iOS）：
1. 已连接WiFi → 点击WiFi名称
2. 查看网络详情中的 **IP地址** 和 **网关**

### 方法2：询问IT部门（推荐）

直接联系公司IT部门或网络管理员，询问：
- ✅ 公司WiFi的IP地址段是什么？
- ✅ 子网掩码是多少？
- ✅ 是否有多个WiFi网络（访客网络、员工网络等）？
- ✅ DHCP分配的IP地址范围是什么？

**示例问题：**
> "你好，我需要配置员工考勤系统的网络验证。请问公司WiFi的IP地址段是什么？比如是 192.168.1.0/24 还是 10.0.0.0/24？"

---

### 方法3：查看路由器管理界面

如果有权限访问公司路由器：
1. 打开浏览器，输入网关地址（通常是 `192.168.1.1` 或 `192.168.0.1`）
2. 登录路由器管理界面
3. 查看 **DHCP设置** 或 **网络设置**
4. 找到 **IP地址池范围**，例如：`192.168.1.100 - 192.168.1.200`

---

## 常见IP地址段说明

### 私有IP地址范围（RFC 1918）：
- **10.0.0.0/8** → `10.0.0.0` 到 `10.255.255.255`（大型企业常用）
- **172.16.0.0/12** → `172.16.0.0` 到 `172.31.255.255`（中型企业）
- **192.168.0.0/16** → `192.168.0.0` 到 `192.168.255.255`（小型企业/家庭常用）

### CIDR表示法说明：
- `/24` = 子网掩码 `255.255.255.0`，覆盖 256 个IP地址（0-255）
- `/32` = 单个IP地址
- `/16` = 子网掩码 `255.255.0.0`，覆盖 65,536 个IP地址

## 配置示例

### 示例1：单个WiFi网络
```yaml
attendance:
  security:
    allowed-networks:
      - 127.0.0.1/32           # 本地开发
      - 192.168.1.0/24         # 公司WiFi（如果IP是192.168.1.x）
```

### 示例2：多个WiFi网络
```yaml
attendance:
  security:
    allowed-networks:
      - 127.0.0.1/32           # 本地开发
      - 192.168.1.0/24         # 员工WiFi
      - 192.168.2.0/24         # 访客WiFi（如果需要）
      - 10.0.0.0/24            # 公司内网
```

### 示例3：精确配置（更安全）
```yaml
attendance:
  security:
    allowed-networks:
      - 127.0.0.1/32           # 本地开发
      - 192.168.1.100/32       # 特定服务器IP
      - 192.168.1.0/24         # 公司WiFi段
```

## 测试配置

配置完成后，测试步骤：

1. **重启后端服务**
2. **查看日志**：尝试打卡时，后端会记录：
   ```
   Validating client IP address: xxx.xxx.xxx.xxx against allowed networks: [...]
   ```
3. **验证结果**：
   - ✅ 如果IP在允许范围内：打卡成功
   - ❌ 如果IP不在允许范围内：会显示错误弹窗，包含实际IP地址

## 注意事项

⚠️ **安全建议：**
- 不要使用太宽泛的IP段（如 `192.168.0.0/16`），这会允许所有私有网络
- 生产环境建议移除 `127.0.0.1/32`（仅用于开发）
- 如果公司有多个办公地点，需要配置所有地点的IP段

⚠️ **常见问题：**
- 如果使用VPN，可能需要配置VPN的IP段
- 如果使用代理服务器，需要配置代理服务器的IP
- 移动网络（4G/5G）的IP通常是公网IP，不在私有IP范围内
